00 Artix Minimal Install & rEFInd Setup

This section walks through a complete command-line installation of Artix Linux (dinit) from the base ISO, removing GRUB entirely and replacing it with rEFInd. By the end you'll have a bootable minimal system ready for the rest of this guide.

ℹ
Assumed setupUEFI system, single NVMe/SSD, GPT partition table. Adapt device paths (nvme0n1, sda, etc.) to match your hardware. Run lsblk to confirm.

Step 1 — Boot the Artix ISO

Download the artix-base-dinit ISO from artixlinux.org, write it to USB, and boot it. You'll land at a root shell. Confirm you're in UEFI mode:

bash
ls /sys/firmware/efi/efivars
# If this directory exists you are in UEFI mode — proceed
# If not, check your BIOS and disable CSM/legacy boot

Step 2 — Connect to the internet

bash — ethernet (auto)
ping -c3 artixlinux.org   # ethernet usually works immediately
bash — wifi (connmanctl)
connmanctl
  enable wifi
  scan wifi
  services                        # lists visible networks
  connect wifi_<your_network_id>
  quit

Step 3 — Partition the disk

We need three partitions: EFI, swap, and root. Adjust sizes to your disk.

bash — identify your disk
lsblk
# Common names: /dev/nvme0n1 (NVMe), /dev/sda (SATA)
bash — partition with fdisk (example: /dev/nvme0n1)
fdisk /dev/nvme0n1

# Inside fdisk:
#  g        — create new GPT partition table
#  n        — new partition
#  1        — partition number
#  (enter)  — default first sector
#  +512M    — EFI partition size
#  t → 1   — set type to EFI System
#  n → 2 → (enter) → +8G   — swap
#  t → 2 → 19              — Linux swap type
#  n → 3 → (enter) → (enter) — root, rest of disk
#  w        — write and exit

Step 4 — Format partitions

bash
# EFI partition
mkfs.fat -F32 /dev/nvme0n1p1

# Swap
mkswap /dev/nvme0n1p2
swapon /dev/nvme0n1p2

# Root (ext4 — or use btrfs if preferred)
mkfs.ext4 /dev/nvme0n1p3

Step 5 — Mount filesystems

bash
mount /dev/nvme0n1p3 /mnt
mkdir -p /mnt/boot/efi
mount /dev/nvme0n1p1 /mnt/boot/efi

Step 6 — Install base system

bash — basestrap (Artix's pacstrap equivalent)
basestrap /mnt \
  base base-devel \
  dinit dinit-rc \
  linux linux-headers linux-firmware \
  amd-ucode \
  networkmanager networkmanager-dinit \
  neovim git curl wget \
  efibootmgr
✦
amd-ucode installs AMD CPU microcode updates. rEFInd will load it automatically as an initrd alongside your main initramfs — no manual configuration needed.

Step 7 — Generate fstab

bash
fstabgen -U /mnt >> /mnt/etc/fstab
cat /mnt/etc/fstab   # verify it looks correct

Step 8 — Chroot into the new system

bash
artix-chroot /mnt

Step 9 — System configuration

bash — locale, timezone, hostname
# Timezone
ln -sf /usr/share/zoneinfo/America/Chicago /etc/localtime
hwclock --systohc

# Locale — uncomment en_US.UTF-8 in /etc/locale.gen
sed -i 's/#en_US.UTF-8/en_US.UTF-8/' /etc/locale.gen
locale-gen
echo "LANG=en_US.UTF-8" > /etc/locale.conf

# Hostname
echo "artix-gaming" > /etc/hostname

# Hosts file
cat >> /etc/hosts <<EOF
127.0.0.1   localhost
::1         localhost
127.0.1.1   artix-gaming.localdomain artix-gaming
EOF

Step 10 — Root password & user

bash
passwd   # set root password

# Create your user account
useradd -mG wheel,audio,video,input,storage -s /bin/bash <username>
passwd <username>

# Allow wheel group to use sudo
sed -i 's/# %wheel ALL=(ALL:ALL) ALL/%wheel ALL=(ALL:ALL) ALL/' /etc/sudoers

Step 11 — mkinitcpio (with early amdgpu)

text — /etc/mkinitcpio.conf
MODULES=(amdgpu)
HOOKS=(base udev autodetect modconf block filesystems keyboard fsck)
bash
mkinitcpio -P

Step 12 — Install rEFInd (skip GRUB entirely)

We install rEFInd directly, bypassing GRUB. rEFInd auto-detects all kernels on the ESP and boot partition — no config file is strictly required to get a working boot.

bash
pacman -S refind
refind-install
# This copies rEFInd to /boot/efi/EFI/refind/ and registers it
# with the UEFI firmware as the default boot entry
⚠
Do NOT install grubNever run pacman -S grub or grub-install — rEFInd handles everything. Having both bootloaders installed can create conflicting UEFI entries.

Step 13 — Configure rEFInd for linux-cachyos + AMD params

rEFInd auto-detects kernels but needs your AMD kernel parameters. Create refind_linux.conf in /boot so rEFInd picks them up automatically for every kernel it finds:

text — /boot/refind_linux.conf
"Boot default"   "root=PARTUUID=<your-partuuid> rw quiet amdgpu.ppfeaturemask=0xffffffff amd_pstate=active iommu=pt"
"Boot fallback"  "root=PARTUUID=<your-partuuid> rw initrd=/boot/initramfs-linux-fallback.img"
bash — get your root PARTUUID
blkid -s PARTUUID -o value /dev/nvme0n1p3
# Paste the output into refind_linux.conf above
✦
amd-ucode autoloadingrEFInd automatically loads /boot/amd-ucode.img as an early initrd when it detects it. No manual initrd line is needed for microcode — it just works.

Step 14 — Enable NetworkManager & reboot

bash
# Enable NetworkManager via dinit
ln -s /etc/dinit.d/NetworkManager /etc/dinit.d/boot.d/

# Exit chroot, unmount, reboot
exit
umount -R /mnt
reboot
ℹ
First bootrEFInd should appear automatically and show your Artix kernel. Select it, boot in, log in as your user, then continue with Section 01 to add CachyOS repos and install the gaming kernel.

01 CachyOS Kernel & System Optimization

Instead of the standard Arch linux or linux-zen kernel, we're going with linux-cachyos — a performance-tuned kernel built specifically for desktop and gaming responsiveness. It ships with the BORE/EEVDF scheduler, AMD-pstate improvements, BBRv3 networking, LTO compilation, and cherry-picked patches that haven't yet landed in mainline.

Step 1 — Full base system update

bash
sudo pacman -Syu
# Reboot if anything was updated before continuing

Step 2 — Add the CachyOS repository

CachyOS provides an automated script that detects your CPU's highest supported x86-64 micro-architecture level (v3, v4, or znver4 for AMD Zen 4/5) and adds the correct optimised repos automatically:

bash — add CachyOS repos (automated)
cd /tmp
curl -O https://mirror.cachyos.org/cachyos-repo.tar.xz
tar xvf cachyos-repo.tar.xz && cd cachyos-repo
sudo ./cachyos-repo.sh   # detects CPU arch, adds correct repos, imports keys
ℹ
What the script doesIt imports the CachyOS GPG keyring, adds the appropriate mirrorlists, and inserts the repo blocks into /etc/pacman.conf above the Arch repos. A backup of your original pacman.conf is saved automatically.

Manual repo config (if you prefer not to run the script)

First check your CPU's highest supported level:

bash — detect x86-64 level
/lib/ld-linux-x86-64.so.2 --help | grep "supported, searched"
# Output example: x86-64-v3 (supported, searched)
# For AMD Zen 4/5, check: gcc -Q --help=target | grep march
text — /etc/pacman.conf (add ABOVE [core], for x86-64-v3 CPUs)
# CachyOS optimised repositories — place above [core]
[cachyos-v3]
Include = /etc/pacman.d/cachyos-v3-mirrorlist
[cachyos-core-v3]
Include = /etc/pacman.d/cachyos-v3-mirrorlist
[cachyos-extra-v3]
Include = /etc/pacman.d/cachyos-v3-mirrorlist
[cachyos]
Include = /etc/pacman.d/cachyos-mirrorlist
✦
AMD Zen 4 / Zen 5 users (Ryzen 7000/9000 series) — use [cachyos-znver4] repos instead of v3/v4. These are compiled with -march=znver4 and offer the best performance on that architecture.

Step 3 — Install linux-cachyos kernel

CachyOS provides several kernel flavours. The default linux-cachyos uses the BORE+EEVDF scheduler — the best choice for gaming and desktop responsiveness:

PackageSchedulerBest for
linux-cachyosBORE + EEVDFGaming, desktop — best all-rounderRecommended
linux-cachyos-boreBOREMaximum interactivity under heavy loadOptional
linux-cachyos-sched-extsched-ext (BPF)Experimental — pluggable userspace schedulersOptional
linux-cachyos-ltsBORE + EEVDFStability-focused, LTS baseOptional
bash — install CachyOS kernel
sudo pacman -Syu   # sync new repos first
sudo pacman -S linux-cachyos linux-cachyos-headers
⚠
Keep a fallback kernelUntil you've confirmed linux-cachyos boots successfully, keep your existing linux kernel installed. rEFInd will show both at boot — you can always fall back.

Step 4 — Update rEFInd stanza for linux-cachyos

The CachyOS kernel uses a different vmlinuz filename. Update your refind_linux.conf:

text — /boot/refind_linux.conf
"CachyOS BORE+EEVDF"  "root=PARTUUID=<your-partuuid> rw quiet amdgpu.ppfeaturemask=0xffffffff amd_pstate=active iommu=pt initrd=/boot/initramfs-linux-cachyos.img"
"Fallback linux"       "root=PARTUUID=<your-partuuid> rw initrd=/boot/initramfs-linux-fallback.img"
ℹ
vmlinuz pathThe CachyOS kernel installs as /boot/vmlinuz-linux-cachyos. If using a manual rEFInd stanza (not auto-scan), update the loader line to this path and set initrd to /boot/initramfs-linux-cachyos.img.

Step 5 — mkinitcpio with amdgpu early KMS

text — /etc/mkinitcpio.conf
MODULES=(amdgpu)
HOOKS=(base udev autodetect modconf block filesystems keyboard fsck)
bash
sudo mkinitcpio -P   # rebuilds ALL kernel presets including linux-cachyos

Step 6 — Update linux-firmware

bash
sudo pacman -S linux-firmware
ls /usr/lib/firmware/amdgpu/ | grep navi3
# Expect: navi32_ce.bin, navi32_me.bin, navi32_pfp.bin, navi32_rlc.bin ...

Step 7 — Gaming sysctl tweaks

text — /etc/sysctl.d/99-gaming.conf
# Keep more in RAM, reduce swap thrash
vm.swappiness                    = 10

# Steam and game engines need large inotify limits
fs.inotify.max_user_watches      = 524288

# Scheduler tuning — less aggressive preemption
kernel.sched_min_granularity_ns  = 500000
kernel.sched_wakeup_granularity_ns = 1000000

# Required by many modern games and Proton (default 65530 is too low)
vm.max_map_count                 = 2147483642
bash — apply immediately
sudo sysctl --system
✦
CachyOS kernel patches relevant to your GPU include cherry-picked AMD-pstate improvements and amdgpu fixes from the mailing list — meaning you often get RDNA3 fixes weeks before they land in the mainline stable kernel.

02 rEFInd Kernel Parameters (AMD)

With rEFInd installed and the base system running, this section covers updating your boot parameters for the CachyOS kernel installed in Section 01. Since rEFInd reads /boot/refind_linux.conf directly at boot, there is no regeneration step — edits take effect on the next reboot.

Update refind_linux.conf for linux-cachyos

Edit /boot/refind_linux.conf to point at the CachyOS initramfs and include all AMD gaming parameters:

text — /boot/refind_linux.conf
"CachyOS BORE+EEVDF"  "root=PARTUUID=<your-partuuid> rw quiet amdgpu.ppfeaturemask=0xffffffff amd_pstate=active iommu=pt initrd=/boot/initramfs-linux-cachyos.img"
"Fallback"            "root=PARTUUID=<your-partuuid> rw initrd=/boot/initramfs-linux-cachyos-fallback.img"
ℹ
vmlinuz filenameThe CachyOS kernel installs as /boot/vmlinuz-linux-cachyos. rEFInd auto-detects it via the kernel auto-scan — you only need to ensure the initrd line in refind_linux.conf matches the correct filename.

AMD kernel parameters reference

ParameterPurpose
amdgpu.ppfeaturemask=0xffffffffUnlocks all power/performance features including manual fan and clock control via CoreCtrl
amd_pstate=activeEnables AMD P-State CPU driver — lower latency frequency scaling on Zen 3+
iommu=ptIOMMU passthrough mode — reduces overhead for GPU gaming workloads

Verify GPU is detected after reboot

bash
lspci -k -d ::0300
# Should show: Kernel driver in use: amdgpu
dmesg | grep amdgpu
uname -r
# Should show: 6.x.x-x-cachyos

04 dinit Services Setup

Unlike systemd, Artix with dinit manages services via /etc/dinit.d/. The key services relevant to GPU gaming are: dbus, polkit, and (if using Wayland compositors) the display server target.

✖
Criticaldbus and polkit MUST be running before launching a desktop session or GPU power management tools. Missing dbus causes silent failures in GameMode and MangoHud.

Enable required base services

bash — dinit service activation
# Enable dbus
sudo dinitctl enable dbus

# Enable polkit (required for user-level GPU power control)
sudo dinitctl enable polkit

# Enable your display manager if using one (e.g. sddm, lightdm)
sudo dinitctl enable sddm

# Start them immediately (without rebooting)
sudo dinitctl start dbus
sudo dinitctl start polkit

Create a custom dinit service for corectrl (optional)

If using CoreCtrl for GPU overclocking, create a user service:

text — /etc/dinit.d/corectrl
type            = process
command         = /usr/bin/corectrl
depends-on      = dbus
depends-on      = polkit
restart         = false
logfile         = /var/log/corectrl.log

Checking service status

bash
dinitctl list
dinitctl status dbus
dinitctl status polkit
✦
dinit user servicesdinit also supports per-user service directories at ~/.config/dinit.d/. Run dinit --user in your session to manage user-scoped services like pipewire or gamemoded.

05 Environment Variables

Set these in your shell profile (~/.bash_profile, ~/.zprofile, or your compositor's exec script):

bash — ~/.bash_profile or ~/.zprofile
# Force RADV (Mesa Vulkan) over AMDVLK if both are installed
export AMD_VULKAN_ICD=RADV

# Enable ACO shader compiler (default in Mesa 22+, explicit for clarity)
export RADV_PERFTEST=aco,nggc

# VA-API driver for hardware video decode
export LIBVA_DRIVER_NAME=radeonsi

# Wayland: use AMD GBM backend
export GBM_BACKEND=amdgpu

# For SDL2-based games: prefer Wayland if on a Wayland compositor
export SDL_VIDEODRIVER=wayland  # remove if using X11

# Disable vsync globally (let games/compositor manage it)
export vblank_mode=0
⚠
SDL_VIDEODRIVEROnly set SDL_VIDEODRIVER=wayland if your session runs a Wayland compositor. On X11 this will break SDL-based games.

08 Performance Tuning & Overclocking

GPU power profile

The amdgpu driver exposes power profiles via sysfs. For maximum gaming performance:

bash — set high performance power profile
# List available profiles
cat /sys/class/drm/card0/device/power_dpm_force_performance_level

# Set high performance
echo "high" | sudo tee /sys/class/drm/card0/device/power_dpm_force_performance_level

# Or use "auto" for balanced (default)
echo "auto" | sudo tee /sys/class/drm/card0/device/power_dpm_force_performance_level

Persist the power profile via dinit oneshot service

text — /etc/dinit.d/amdgpu-perf
type        = scripted
command     = /usr/local/bin/amdgpu-perf.sh
depends-on  = udev
bash — /usr/local/bin/amdgpu-perf.sh
#!/bin/sh
echo "high" > /sys/class/drm/card0/device/power_dpm_force_performance_level
echo "5" > /sys/class/drm/card0/device/pp_power_profile_mode
# Profile 5 = "VR" mode — high sustained clocks, good for gaming
bash
sudo chmod +x /usr/local/bin/amdgpu-perf.sh
sudo dinitctl enable amdgpu-perf
sudo dinitctl start amdgpu-perf

CoreCtrl (GUI overclock tool)

bash
paru -S corectrl

# Allow user to control GPU without sudo (polkit rule)
sudo cp /usr/share/polkit-1/rules.d/90-corectrl.rules \
         /etc/polkit-1/rules.d/
⚠
Kernel parameter requiredCoreCtrl's full GPU control requires amdgpu.ppfeaturemask=0xffffffff in your kernel command line (set in Section 1).

# OpenGL renderer

Common issues

✖
Black screen / no displayEnsure amdgpu is in your initramfs. Add it to MODULES=(amdgpu) in /etc/mkinitcpio.conf, then run mkinitcpio -P.
✖
Games crash with Vulkan errorsCheck that vulkan-radeon ICD is being picked up: VK_ICD_FILENAMES=/usr/share/vulkan/icd.d/radeon_icd.x86_64.json vulkaninfo
⚠
GameMode not activatingConfirm gamemoded is running (dinitctl --user status gamemoded) and your user is in the gamemode group (groups $USER).
✦
dmesg GPU errorsIf you see amdgpu: failed to allocate VRAM errors, add amdgpu.gartsize=512 to kernel params. For firmware errors, ensure linux-firmware is fully up to date.

Install radeontop for live GPU monitoring

bash
sudo pacman -S radeontop
radeontop  # live GPU utilization, VRAM, clocks

11 Hyprland Rice — Basic Setup

Hyprland is a dynamic tiling Wayland compositor with a custom renderer supporting animations, rounded corners, and blur. It runs natively on AMD/RADV with zero special configuration needed — unlike Nvidia. This section covers a clean base rice: Hyprland + Waybar + hyprpaper + rofi + dunst + a colour theme.

✦
AMD advantageHyprland officially states NVIDIA support is unsupported. AMD with RADV is the cleanest, lowest-friction Hyprland experience available.

  noto-fonts-emoji \
  grim slurp \
  wl-clipboard \
  brightnessctl \
  playerctl

Launch Hyprland from TTY (dinit-compatible)

Since you're using dinit (not systemd), the cleanest way to start Hyprland is directly from TTY1 via your shell profile — no display manager required:

bash — ~/.bash_profile or ~/.zprofile
# Auto-launch Hyprland on TTY1 login
if [[ -z "$WAYLAND_DISPLAY" ]] && [[ "$(tty)" == "/dev/tty1" ]]; then
    export XDG_CURRENT_DESKTOP=Hyprland
    export XDG_SESSION_TYPE=wayland
    export XDG_SESSION_DESKTOP=Hyprland
    export QT_QPA_PLATFORM=wayland
    export MOZ_ENABLE_WAYLAND=1
    exec Hyprland
fi
ℹ
SDDM (optional)If you prefer a display manager, sddm works flawlessly with Hyprland. Install it, enable it in dinit (dinitctl enable sddm), and select the Hyprland session at login.

dinit user service — start Hyprland support daemons

Create a user dinit service to ensure the XDG portal and polkit agent start with your session:

text — ~/.config/dinit.d/hyprland-session
type       = scripted
command    = /usr/lib/xdg-desktop-portal-hyprland
depends-on = dbus
